# 厨房陪我 MVP 信息架构与页面规格

## 1. 文档信息

- 版本：v0.1
- 日期：2026-02-21
- 关联：
  - `docs/prd/core/2026-02-21-kitchenwithme-mvp-prd.md`
  - `docs/prd/design/2026-02-21-kitchenwithme-mvp-figma-spec.md`

## 2. 信息架构（MVP）

```text
首页（今日开火）
├─ 选已有菜（快速选择）
│  └─ 进入做饭中会话
├─ 三餐日历（历史入口）
│  └─ 记录详情
├─ 味觉记忆（列表）
│  └─ 记忆详情
└─ 冰箱角落（MVP 仅展示入口，不扩展复杂功能）
```

## 3. 全局导航规则

- 顶部主导航固定 4 项：今日开火 / 三餐日历 / 味觉记忆 / 冰箱角落
- 首屏默认聚焦“今日开火”，其余入口不抢主任务
- 所有页面保留“返回今天做饭流程”的快捷入口

## 4. 页面规格

## 4.1 Screen: Start-Home-Default

- 关联需求：FR-001
- 页面目标：把用户快速带入“开始做饭”
- 入口：首次打开、完成一次流程后返回
- 出口：进入选菜页、进入历史模块
- 必备元素：
  - 今日开火主卡（主 CTA）
  - 最近一次味觉记忆摘要
  - 当周开火次数轻提示
- 文案语气：建议式，不命令
- 成功指标：点击今日开火按钮转化率

## 4.2 Screen: Start-RecipePicker-List / Selected

- 关联需求：FR-002
- 页面目标：2 步内从已有菜进入会话
- 入口：首页 CTA
- 出口：进入 Cooking-Session-Idle
- 必备元素：
  - 已有菜列表
  - 菜谱难度/时长轻标签
  - 已选中态与“开始陪我做饭”按钮
- 状态：
  - 默认未选
  - 已选可进入
- 异常：列表为空时展示温柔兜底文案与返回首页

## 4.3 Screen: Cooking-Session-Idle / Listening / Speaking / Interrupted / TimerActive

- 关联需求：FR-003 / FR-004 / FR-005 / FR-006 / FR-009
- 页面目标：完成做饭中连续陪伴
- 入口：选菜后开始会话
- 出口：进入 Reflection-Entry-Default
- 必备元素：
  - 当前步骤卡片（步骤编号、内容、关键提醒）
  - 语音光晕球（听/说/中断状态）
  - 快捷指令提示（下一步、重复、慢一点、计时）
  - 食材替代建议入口
- 状态定义：
  - Idle：会话初始化
  - Listening：聆听用户
  - Speaking：系统陪伴回复
  - Interrupted：用户打断后恢复
  - TimerActive：存在运行中计时
- 边界处理：
  - 麦克风拒绝 -> 点按替代模式
  - 弱网 -> 文字+步骤降级
  - 识别失败 -> 引导短指令重试
- 成功指标：会话完成率、指令成功理解率、兜底触发率

## 4.4 Screen: Reflection-Entry-Default / Filled / Success

- 关联需求：FR-007
- 页面目标：30 秒内完成轻记录
- 入口：会话完成
- 出口：返回首页或历史页
- 必备元素：
  - 味觉印象输入
  - 心情输入/标签
  - 提交按钮（轻语气）
- 状态：
  - 默认空态
  - 已填写可提交
  - 提交成功反馈
- 成功指标：记录提交率、平均填写时长

## 4.5 Screen: History-Calendar-Week / Memory-List / Memory-Detail

- 关联需求：FR-008
- 页面目标：回看开火记录与味觉记忆
- 入口：首页导航、记录成功后跳转
- 出口：返回首页或再次开火
- 必备元素：
  - 周视图日历与开火标记
  - 当日记录列表
  - 记录详情（菜名、味觉、心情、时间）
- 成功指标：历史回看率、二次开火转化

## 4.6 Screen: Edge-MicPermission-Denied / Network-Unstable / STT-Failed / TTS-Failed

- 关联需求：FR-009
- 页面目标：异常情况下保持陪伴不断流
- 必备元素：
  - 异常说明（非告警式）
  - 可执行下一步操作
  - 回归主链路按钮
- 文案要求：先安抚，再给操作

## 5. 数据展示与空态原则

- 空态不使用“失败/错误”语气，使用“还没有，等你点亮”语气
- 无记录时展示“如何开始一次最小会话”的引导
- 历史数据优先显示最近 7 天

## 6. 视觉与交互红线

- 不出现进度条、对勾、叉号、任务完成语义
- 不出现命令式词汇（如“立即”“必须”“马上”）
- 不使用纯黑 #000000 或高饱和警示色作为主视觉

## 7. 页面评审清单

- 是否聚焦“做饭中陪伴”主价值
- 是否 2 步进入会话
- 是否支持打断后恢复
- 是否 30 秒内可完成开火后记录
- 是否所有异常都能回归主链路
