# 厨房陪我 MVP PRD（Web）

## 1. 文档信息

- 版本：v0.1
- 日期：2026-02-21
- 产品阶段：MVP
- 关联设计文档：`docs/prd/plans/2026-02-21-kitchenwithme-mvp-design.md`

## 2. 背景与问题

### 2.1 背景

用户愿意做饭，但在日常高压节奏中，做饭容易变成负担。现有工具多数强调效率和管理，缺乏“被温柔陪伴”的体验。

### 2.2 需要解决的问题

- 做饭中容易手忙脚乱，缺少自然引导
- 做完后缺少轻量记录，记忆无法沉淀
- 开始做饭前决策成本高

## 3. 目标与非目标

### 3.1 MVP 目标

- 提供做饭中连续语音陪伴（主价值）
- 提供做饭后轻记录能力（味觉+心情）
- 提供开火前已有菜快速选择

### 3.2 非目标（MVP 不做）

- 复杂营养分析与健康评分
- 打卡/KPI/排行榜等任务化机制
- 社交内容流与分享增长玩法

## 4. 目标用户

- 城市白领（20-40 岁）：节奏快，想做饭但怕麻烦
- 烹饪爱好者：愿意做饭，希望记录味觉与生活感受

## 5. 品牌约束（必须满足）

- 文案禁忌：任务、打卡、KPI、管理等工具化词汇
- 语气禁忌：命令、催促、评判、教训
- 视觉禁忌：纯黑、高饱和、硬拐角、拟人卡通符号
- 状态表达：使用柔光/亮点，不用对勾/叉号/进度条

## 6. MVP 范围

### 6.1 P0：做饭中语音陪伴

- 功能点
  - 开始做饭会话
  - 连续语音收发（可打断）
  - 步骤导航：下一步/重复/慢一点
  - 计时器：开始、提醒、冲突处理
  - 食材替代与火候问答（限定域）
  - 异常降级：弱网/识别失败/播放失败

- 验收标准
  - 用户可完成一个完整菜谱关键步骤
  - 关键指令响应稳定可用
  - 异常场景有明确兜底路径

### 6.2 P1：开火后轻记录

- 功能点
  - 记录味觉印象（短文本）
  - 记录心情（轻量标签/文本）
  - 保存到“味觉记忆”并可回看

- 验收标准
  - 记录操作 <= 30 秒完成
  - 可在历史中查看当日记录

### 6.3 P2：开火前已有菜选择

- 功能点
  - 展示已有菜列表
  - 快速进入做饭中会话

- 验收标准
  - 2 步内进入会话

## 7. 用户故事

1. 作为一个下班后做饭的人，我希望在做饭时被温柔引导，这样我不会慌乱。
2. 作为一个爱做饭的人，我希望做完后快速记下味道和心情，方便以后回看。
3. 作为一个不想纠结吃什么的人，我希望开火前快速从已有菜中选一个开始。

## 8. 关键流程

1. 首页 -> 今日开火
2. 选择已有菜 -> 进入做饭中会话
3. 语音陪伴完成关键步骤
4. 完成后进入轻记录
5. 回到首页或历史页查看记录

## 9. 功能需求明细

| 编号 | 模块 | 需求 | 优先级 |
| --- | --- | --- | --- |
| FR-001 | 今日开火 | 展示“开始做饭”入口并默认聚焦 | P0 |
| FR-002 | 已有菜选择 | 支持从已有菜列表快速选择 | P2 |
| FR-003 | 语音会话 | 支持连续双向语音和中断 | P0 |
| FR-004 | 步骤控制 | 支持下一步/重复/慢一点 | P0 |
| FR-005 | 计时能力 | 支持创建和提醒计时器 | P0 |
| FR-006 | 替代建议 | 提供食材替代建议 | P0 |
| FR-007 | 开火后记录 | 支持味觉+心情轻记录 | P1 |
| FR-008 | 历史回看 | 三餐日历可查看记录摘要 | P1 |
| FR-009 | 异常兜底 | 权限、弱网、识别失败可降级 | P0 |

## 10. 非功能需求

- 性能
  - 语音响应延迟 P50 < 1.2s，P95 < 2.0s
  - 中断响应 < 400ms
- 可用性
  - 主链路完成率 > 60%
- 品牌一致性
  - 核心页面无禁忌词与任务符号
- 安全与合规
  - 明确麦克风权限用途
  - 提供语音数据处理说明

## 11. 数据与埋点（最小集）

- 会话维度
  - session_start, session_end, session_duration
- 语音维度
  - stt_latency, tts_latency, interruption_count, fallback_count
- 业务维度
  - recipe_selected, step_completed, reflection_saved

## 12. 发布门槛（Go / No-Go）

- P0/P1 功能通过 E2E 验收
- 核心异常路径全覆盖测试通过
- 品牌文案审校通过
- 指标面板可查看核心埋点

## 13. 风险与缓解

- 风险：Web 连续语音体验不稳定
  - 缓解：先 B 方案，保留 A 升级接口
- 风险：识别错误影响体验
  - 缓解：有限域语义 + 温柔兜底
- 风险：范围膨胀
  - 缓解：严格执行 P0/P1/P2 和非目标
